
## TraceMate Developer Specification

A comprehensive guide for frontend, backend, and database setup of the TraceMate drawing-overlay app.

---

# 1. App Overview & Personas

**TraceMate** is a mobile-first React + TypeScript drawing-overlay tool. Users upload an image, overlay it on live camera feed, and trace in real time.

**Target Users:**

* Kids learning to draw
* Beginners practicing tracing
* Hobbyists looking for a fun sketch tool
* Social sharers who want interesting content

**Key Technologies:** React, TypeScript, Tailwind CSS, Framer Motion, Supabase (Auth, RLS, Policies), lightweight video formats, manual-payment integration.

---

# 2. Payment & Plans

## 2.1. Manual Payment Flow

* No automated gateway (e.g., Stripe).
* User clicks **Upgrade** → shows WhatsApp/RedotPay phone number & instructions.
* Admin manually processes payment, creates user account, assigns paid **role** in Supabase.

## 2.2. Pricing Tiers

| Plan | Usage Limits                      | Sign-in Required? | Landing CTA   |
| ---- | --------------------------------- | ----------------- | ------------- |
| Free | 1 min/session; 5 sessions/day     | No                | Try TraceMate |
| Paid | Unlimited overlay time & sessions | Yes               | Go to App     |

* **Landing Page**: If user is signed-in & paid, only **Go to App** is shown.

# 7. Tracing Page (Core Feature)

1. **Camera Access Prompt:**

   * `navigator.mediaDevices.getUserMedia()` requested once per session.
   * If denied → instructions card with “Enable camera” link.

2. **Live Feed + Overlay:**

   * `<video>` fills screen; `<canvas>` overlays user image.

3. **Settings Panel:** Slide-in drawer containing:

   * **Opacity Slider** (0–100%)
   * **Scale Slider** (0.1×–3×)
   * **Rotation Slider** (–180° to +180°)
   * **Corner Transform Sliders** for 4-point perspective adjustments.
   * **Camera Switch** button (front/back).

4. **Touch Gestures:**

   * Pinch-to-zoom & drag overlay.
   * Auto-hide sliders after 3 s inactivity; tap to reveal.

5. **Controls:**

   * **X‑Button:** exit tracing → App Main.
   * *(Future)* Snapshot capture of canvas + video frame.

6. **Mobile-first UI:**

   * Sliders & buttons sized for touch, minimal overlap with video.

# 8. Technical & UI Notes

* **Frameworks:** React + TypeScript, functional components & hooks.
* **Styling:** Tailwind CSS, with mobile breakpoints (`sm`, `md`, etc.).
* **Animations:** Framer Motion for text, panel transitions.
* **Backend:** Supabase for Auth, RLS, Policies, user metadata.
* **Performance:**

  * Compress videos (H.264), lazy-load with `loading="lazy"`.
  * Serve multiple resolutions via `<source>`.
* **Accessibility:**

  * ARIA labels on buttons & sliders.
  * Keyboard navigation & high-contrast mode (roadmap).

> **Upcoming Change:** In Q2 2025, keys will rename to `publishable` & `secret`.

## 9.2. SQL Schema

```sql
-- 1. Users table
CREATE TABLE public.users (
  id             uuid         PRIMARY KEY DEFAULT gen_random_uuid(),
  email          text         UNIQUE NOT NULL,
  password_hash  text         NOT NULL,
  role           text         NOT NULL DEFAULT 'free',
  created_at     timestamp    NOT NULL DEFAULT now()
);

-- 2. Usage sessions
CREATE TABLE public.usage_sessions (
  id              uuid      PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid      NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  started_at      timestamp NOT NULL DEFAULT now(),
  duration_secs   integer   NOT NULL,
  completed       boolean   NOT NULL DEFAULT false
);
CREATE INDEX ON public.usage_sessions (user_id, date_trunc('day', started_at));

-- 3. Payments
CREATE TABLE public.payments (
  id              uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid        NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  method          text        NOT NULL,
  reference       text        NOT NULL,
  status          text        NOT NULL DEFAULT 'pending',
  created_at      timestamp   NOT NULL DEFAULT now(),
  updated_at      timestamp   NOT NULL DEFAULT now()
);
```

> **Developer Notes:**
>
> * Enable RLS on each table; define policies so `anonpublic` only inserts into `usage_sessions` and reads public info.
> * Use `service_rolesecret` in server functions to manage `payments` and user roles.
